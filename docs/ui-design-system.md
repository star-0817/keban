# 课伴 UI 设计系统

## 设计定位

课伴面向学生和班委，是 Android 优先、离线优先的校园生产力工具。界面采用现代、简洁、友好的扁平化风格，强调清晰信息层级、舒适留白、低干扰反馈和本机数据边界。

本次使用 `ui-ux-pro-max` 生成设计建议后，保留其 Android 可读性、触控尺寸、移动端安全区和语义化令牌要求；由于工具默认推荐偏应用商店展示和拟物风格，实际落地按 Issue 指定方向调整为青绿扁平校园工具，不采用拟物、渐变、玻璃拟态或厚重阴影。

## 色彩

- 主色：`#0D9488`，用于主按钮、激活导航、关键状态。
- 辅色：`#14B8A6`，用于辅助强调和轻量状态。
- 点缀橙：`#EA580C`，用于小标签、重点提示，不用于大面积背景。
- 背景：`#F0FDFA`，提供浅薄荷白基底。
- 深文字：`#134E4A`，用于标题和正文主要内容。
- 表面：白色和 `#F8FFFD`，用于卡片、列表项、输入容器。
- 边框：`#D5F7EF` 和 `#99F6E4`，轻量分隔，不依赖重阴影。
- 反馈：成功、警告、危险均使用语义令牌和浅色背景配对，避免只靠颜色表达。

页面应优先使用 `src/styles/tokens.css` 中的 `--kb-*` 变量，不在组件中随意写新的十六进制颜色。

## 字体与层级

- 字体栈：系统中文字体优先，保持 Android 和桌面预览一致的高可读性。
- 标题：`--kb-font-title`，用于页面主标题。
- 正文：`--kb-font-body` 与 `--kb-font-body-large`，正文行高保持约 1.45-1.55。
- 辅助文字：`--kb-font-caption`，仅用于标签、状态和说明。
- 中文内容默认换行，不使用过窄容器截断核心文字。

## 间距、圆角与阴影

- 间距采用 4/8dp 思路映射到 rpx：`8rpx`、`12rpx`、`16rpx`、`24rpx`、`32rpx`、`40rpx`。
- 页面横向留白使用 `--kb-space-page-x`，底部内容预留固定导航和安全区。
- 卡片圆角 `24rpx`，控件圆角 `18rpx`，避免过度圆润。
- 阴影只用于卡片和底部导航的轻微层级，不使用厚重投影。

## 组件状态

- 主要按钮使用主色底和白字，最小触控区域 `96rpx`。
- 次要按钮和分段控件使用浅表面、边框和激活态区分。
- 输入框最小高度 `96rpx`，使用可见背景和边框。
- 错误、成功、警告提示使用语义背景，文字说明必须明确。
- 删除操作保留二次确认，危险操作使用危险色文字。

## 素材用途

- `app-icon-1024.png`：App 图标、关于页展示，并尝试接入 Android 图标配置。
- `home-campus-companion.webp`：首页主视觉。
- `local-data-safe.webp`：首页本地保存与隐私说明。
- `empty-class-roster.webp`：班级名单空状态。
- `empty-study-schedule.webp`：学习课程和日程空状态。
- `feature-documents.webp`：首页文档工具开发中占位。
- `feature-coming-soon.webp`：保留为后续开发中占位素材。
- `wx.jpg`：感谢支持页微信收款码静态展示。
- `zfb.jpg`：感谢支持页支付宝收款码静态展示。

图片按页面需要引用，不做全局预加载。

## 无障碍与移动端要求

- 底部导航和主要按钮触控区域约 48dp。
- 固定底部导航下方内容需保留安全区和滚动底部间距。
- 不使用表情符号作为结构性图标；当前导航使用统一字母符号占位，后续可替换为同风格 SVG。
- 文字和状态说明不只依赖颜色传达。
- 支持页不弹窗、不强制支付，不接支付 SDK，不读取用户支付信息。

## Android 图标说明

`src/manifest.json` 已将 `app-icon-1024.png` 写入 `app-plus.icon` 和 Android `distribute.android.icons`。不同 UniApp/HBuilderX 构建链对图标字段支持存在差异，是否正确生成各密度启动图标需要在真机安装包中验证；如不生效，应在 HBuilderX 可视化 manifest 或平台图标生成流程中补齐密度资源。
