# Android SQLite 持久化真机验证

本任务接入的真实 SQLite 只在 Android App 且 `plus.sqlite` 可用时启用。H5、Vitest、Node、浏览器预览和缺少 `plus.sqlite` 的运行环境会降级为内存预览数据库，页面提示为“当前为本地临时数据，重启后会清空。”。

## 测试环境与真机环境区别

- 自动化测试使用 `src/repositories/testing/inMemorySqliteDatabase.ts`，只验证纯 TypeScript 运行时选择、迁移幂等、仓储共享和中文错误信息。
- Android 真机使用 `src/plugins/sqlite.ts` 中的 `plus.sqlite` 适配器，数据库名固定为 `keban.db`，路径为 `_doc/keban.db`。
- UniApp SQLite API 当前没有在类型定义中暴露绑定参数能力，因此参数在插件层统一进行严格序列化：字符串单引号转义为两个单引号，`null` 转为 `NULL`，数字必须是有限值。仓储层仍只能使用 `?` 参数，不允许拼接用户输入。
- 如果 `plus.sqlite` 不存在、打开失败或迁移失败，运行时会降级为内存数据库，并保留中文降级原因；这不是持久化模式。

## 首次安装验证

1. 在 Android 真机全新安装调试包。
2. 打开 App，进入“班级工具”。
3. 确认页面提示为“数据仅保存在本机，重启后仍会保留。”。
4. 新增学生：姓名“张三”，学号“2026001”，备注“班长”。
5. 进入“学习”。
6. 新增课程：课程“高等数学”，周一，第 1-2 节，学期“2026-fall”。
7. 新增日程：标题“期中考试”，填写开始和结束 UTC ISO 时间。
8. 确认学生、课程、日程均能立即查询显示。

## 重启后持久化验证

1. 从系统最近任务中强制关闭 App。
2. 重新打开 App。
3. 进入“班级工具”，确认“张三”仍存在。
4. 进入“学习”，确认“高等数学”和“期中考试”仍存在。
5. 重复关闭并重启一次，确认数据仍存在。

## 迁移幂等验证

1. 在已有数据的 App 上连续启动两次。
2. 确认页面没有出现重复学生、重复课程或重复日程。
3. 如可通过调试工具检查数据库，确认 `schema_version` 中每个版本只有一条记录。

## 失败与降级验证

1. 使用临时调试包模拟 `plus.sqlite.openDatabase` 失败，或临时加入失败迁移后构建调试包。
2. 启动 App。
3. 预期页面不崩溃，班级和学习页面提示“当前为本地临时数据，重启后会清空。”。
4. 预期错误或日志中包含中文上下文，例如“Android SQLite 打开失败”或“数据库初始化失败：迁移 X（名称）执行失败”。
5. 在降级状态新增学生、课程和日程，重启后这些降级内存数据会清空。

## 卸载行为

1. 在持久化模式下新增学生、课程和日程。
2. 从系统设置中卸载 App。
3. 重新安装 App。
4. 预期 `_doc/keban.db` 随 App 数据一起被清除，学生、课程和日程不再存在。
