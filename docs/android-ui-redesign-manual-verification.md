# Android UI 重构手工验证清单

## 环境

- 设备：Android 真机或模拟器。
- 构建：执行 `npm run build:android` 后安装调试包。
- 网络：建议分别在联网和断网状态下打开一次。

## 导航与首页

1. 打开 App，确认首页显示 `home-campus-companion.webp` 主视觉。
2. 点击底部“首页、班级、学习、文档、我的”，确认激活态清晰且没有遮挡内容。
3. 点击首页“班级工具”和“学习工具”，确认进入对应页面。
4. 点击首页“文档工具”或底部“文档”，确认仅显示“开发中”提示，不进入未完成业务。
5. 确认首页 `local-data-safe.webp` 正常展示，本地保存说明可读。

## 班级工具

1. 首次无名单时，确认显示 `empty-class-roster.webp` 空状态。
2. 新增学生，确认输入框、按钮和成功后的列表显示正常。
3. 粘贴名单文本并解析预览，确认有效、跳过、错误行仍可查看。
4. 确认导入名单后，执行随机抽人。
5. 切换分组，分别使用“指定组数”和“每组最大人数”生成分组。
6. 删除学生时确认出现二次确认，取消不会删除，确认后才删除。
7. 如 SQLite 不可用，确认临时数据提示仍显示且文案准确。

## 学习工具

1. 课程表为空时，确认显示 `empty-study-schedule.webp` 空状态。
2. 新增课程，切换星期和学期标识，确认课程列表可刷新。
3. 删除课程时确认出现二次确认。
4. 切换日程，新增日程并确认列表显示。
5. 日程为空时，确认显示学习空状态图。
6. 切换绩点，添加、删除课程行并计算绩点。
7. 切换考勤，添加、删除考勤行并计算出勤率。
8. 如 SQLite 不可用，确认临时数据提示仍显示且文案准确。

## 我的 / 感谢支持

1. 点击底部“我的”，确认进入“感谢支持”页面。
2. 确认页面说明“完全自愿、不影响功能、不接支付 SDK、不读取支付信息”。
3. 确认 `wx.jpg` 和 `zfb.jpg` 以小卡片显示。
4. 分别点击两张收款码，确认只打开本机图片预览且二维码清晰。
5. 返回页面后，确认班级和学习页面仍可正常进入。

## App 图标与素材

1. 安装完成后查看桌面图标，确认是否使用 `app-icon-1024.png`。
2. 如桌面图标未生效，记录构建工具版本，并在 HBuilderX/UniApp 图标生成流程中补齐各密度图标。
3. 检查页面图片没有拉伸变形、黑屏或加载失败。

## 视觉与安全区

1. 在 360dp 宽度附近设备查看，确认没有水平滚动和文字互相遮挡。
2. 在手势导航设备查看，确认底部导航不遮挡列表最后一项。
3. 调大系统字体后，确认主要按钮和核心文案仍可读。
4. 横屏打开主要页面，确认内容可滚动且操作控件没有重叠。
