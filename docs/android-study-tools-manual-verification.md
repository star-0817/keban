# Android 学习工具手工验证步骤

适用范围：GitHub Issue #11 学习页面、课程表、日程、绩点和考勤。

1. 全新安装并启动 App，进入首页。
   预期：首页正常显示，底部导航包含“首页、班级、学习、文档、我的”。
2. 点击底部“学习”。
   预期：进入学习页面，顶部显示“学习”，页面显示“当前为本地临时数据，重启后会清空。”。
3. 点击底部“首页”和“班级”，再返回“学习”。
   预期：首页、班级和学习均可互相进入；点击“文档”“我的”仍提示“功能开发中”。
4. 在“课程表”选择周一、学期 `2026-fall`，新增课程：高等数学、王老师、A101、1-2 节、08:00-09:40。
   预期：课程出现在列表中，按开始节次排序。
5. 编辑该课程，将地点改为 B201 并保存。
   预期：列表中的地点更新为 B201。
6. 再新增同一星期、同一学期且节次重叠的课程。
   预期：页面显示中文课程冲突错误。
7. 删除课程。
   预期：先出现二次确认；确认后课程从列表消失，空状态为中文。
8. 在“日程”保留默认 UTC 范围，新增日程：期中考试、`2026-07-23T01:00:00.000Z` 至 `2026-07-23T02:00:00.000Z`、A101、带学生证。
   预期：日程出现在列表中，按开始时间排序。
9. 编辑该日程，将备注改为“提前十分钟到场”并保存。
   预期：列表备注更新。
10. 再新增与该日程时间重叠的日程。
    预期：页面显示中文日程冲突错误。
11. 删除日程。
    预期：先出现二次确认；确认后日程从列表消失。
12. 在“绩点”保留示例行，新增一行等级制课程，选择“及格”，点击“计算绩点”。
    预期：展示总学分、加权绩点、纳入计算课程数和课程明细。
13. 将某行学分改为 0 后点击“计算绩点”。
    预期：显示领域规则返回的中文错误，不写入数据库。
14. 在“考勤”录入两条记录：同一课程分别为“出勤”和“迟到”，UTC 时间分别在 2026-07。
    预期：展示总记录数、折算出勤次数、出勤率和状态次数。
15. 切换“按课程”和“按月”后分别点击“计算考勤”。
    预期：分组结果分别按课程标识和月份显示。
16. 将考勤 UTC 时间清空，选择“按月”计算。
    预期：显示领域规则返回的中文 UTC 时间错误。
17. 强制关闭并重启 App，再进入“学习”。
    预期：此前课程和日程临时数据清空，页面仍显示临时数据提示。

当前限制：本任务未接入真实 Android SQLite、系统通知或日历同步；课程和日程使用明确标识的内存预览数据库，重启后会清空。绩点和考勤只做临时表单计算，不持久化。
