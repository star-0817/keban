# 课程表与日程数据约定

## 课程节次

- 课程按学期标识、星期和节次保存。
- 星期使用 `1` 到 `7`，分别表示周一到周日。
- 节次必须是正整数，`endSlot` 不能小于 `startSlot`。
- 课程时间使用 `HH:mm`，例如 `08:00`；结束时间必须晚于开始时间。
- 同一学期、同一星期且节次区间重叠时视为冲突。`1-2` 与 `3-4` 不冲突，`1-2` 与 `2-3` 冲突。

## 日程 UTC 时间

- 日程开始和结束时间只保存 UTC ISO 字符串，例如 `2026-07-23T01:00:00.000Z`。
- 日程结束时间必须晚于开始时间。
- 日程冲突按 UTC 时间区间判断。一个日程的结束时间等于另一个日程的开始时间时不冲突。
- 当前领域层和仓储层不做跨时区显示转换，页面层后续接入时再负责本地显示格式。

## Android 真机 SQLite 验证步骤

当前任务只实现 SQLite 抽象仓储和测试替身，没有接入真实 UniApp SQLite API。后续接入 Android 真机 SQLite 适配器后，请验证：

1. 全新安装 App 并触发数据库初始化，确认 `schema_version`、`courses`、`schedule_events` 表创建成功。
2. 重启 App 后再次初始化，确认版本 `1`、`2`、`3` 不重复执行，业务表无重复写入。
3. 创建课程，重启后按学期和星期查询，确认课程仍存在并按开始节次升序。
4. 创建同学期、同星期、重叠节次课程，确认返回“课程时间与已有课程冲突”。
5. 创建日程，重启后按 UTC 范围查询，确认日程仍存在并按开始时间升序。
6. 创建 UTC 时间重叠日程，确认返回“日程时间与已有日程冲突”。
7. 创建边界相接的课程和日程，确认课程 `1-2` 与 `3-4` 可保存，日程结束等于另一个开始时可保存。
