# Android 班级工具手工验证步骤

适用范围：GitHub Issue #7 班级名单、抽人和分组页面。

1. 全新安装并启动 App，进入首页。
   预期：首页正常显示，底部导航包含“首页、班级、学习、文档、我的”。
2. 点击底部“班级”。
   预期：进入班级工具页面；Android 真机 SQLite 可用时显示“数据仅保存在本机，重启后仍会保留。”，降级环境显示“当前为本地临时数据，重启后会清空。”。
3. 在“名单”页新增学生“张三”，学号“2026001”，备注“班长”。
   预期：名单出现张三，总人数变为 1。
4. 编辑张三，将备注改为“语文课代表”。
   预期：列表中备注更新。
5. 删除张三。
   预期：出现二次确认；确认后名单为空，总人数为 0。
6. 粘贴导入文本：
   ```text
   姓名,学号,备注
   李四,2026002,组长
   ,2026003,缺少姓名
   王五,2026004,
   ```
   点击“解析预览”。
   预期：显示有效 2 条、错误 1 行；错误行不会写入名单。
7. 点击“确认导入”。
   预期：名单新增李四和王五，总人数为 2。
8. 切换“抽人”，选择抽取 1 人，并勾选排除李四。
   预期：结果只可能包含王五，显示种子，点击复制可复制种子。
9. 切换“分组”，选择“指定组数”并输入 2。
   预期：显示 2 个分组，每名学生只出现一次，显示种子。
10. 点击底部“首页”。
    预期：返回首页；点击“学习、文档、我的”仍提示“功能开发中”。

SQLite 持久化、重启恢复、失败降级和卸载清除验证见 [Android SQLite 持久化真机验证](android-sqlite-persistence-verification.md)。
